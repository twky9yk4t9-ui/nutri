# NUTRI

Personal meal operating system — a phone-first, offline PWA that runs a fixed
food system: permanent breakfast, an 8-template snack canon (max one Skyr-based
snack per day, ≥20 g combined snack protein), 8 main recipes, a 4-session
weekly cook cycle (Sat→Fri), one weekly shop, tap-to-log adherence, morning
weight. Single user, no backend, no accounts, no network calls. Full product
spec: [nutrition-app-spec.md](nutrition-app-spec.md).

## Commands

```bash
npm install
npm run dev        # dev server (localhost:5173)
npm test           # 78 unit tests: macros seed guard, scaling, rotation, grocery, dates, adherence, backup/migration
npm run build      # type-check + production build + service worker (dist/)
npm run preview    # serve the production build
npm run icons      # regenerate public/*.png from scripts/make-icons.mjs (zero-dep)
```

The PWA layer is plugin-free: `public/manifest.webmanifest` is static, the
service worker is generated by `scripts/build-sw.mjs` (zero-dep, runs as the
last build step, precaches every asset with content-hash revisions), and
registration is three lines in `src/main.tsx`.

## Install on a phone

Serve `dist/` over HTTPS (or localhost), open it in Safari/Chrome, then
"Add to Home Screen". After the first load the app is fully offline.

## Where things live

- `src/domain/` — pure logic, no React: types (§8), seed data (§3–§5, §10),
  cycle/rotation/snack rules (§6), grocery aggregation (§6.3), scaling (§5.1),
  adherence & weights (§7). Everything here is unit-tested.
- `src/state/` — localStorage persistence (key `nutri:v1`, debounced writes,
  schema version for migrations, JSON export/import) and the React store.
- `src/screens/` — the six screens (§9).
